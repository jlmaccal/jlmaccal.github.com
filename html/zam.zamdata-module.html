Module zamdata

source code

Major ZAM data structures.

This module implements three major ZAM data structures:

  • ZamDataClass (master ZAM database)
  • FragDataClass (structure describing a ZAM fragment)
  • SnapDatabaseClass (database of snapshots or structures)

The ZamDataClass is the master ZAM datastructure and contains information such as the sequence of the target and the current stage of the ZAM run. It also maintains a list of fragments. There is usually only one instance of the class at a time and it is usually named zd by convention. zd is also the object that is cPickled to disk as 'restart.dat'.

The FragDataClass represents a ZAM fragment. It is the most complicated data structure in ZAM and represents a fragment. It contains information such as the sequence, length, and parents, as well as run parameters, analysis results, and much more. Much of ZAM revolves around creating or manipulating the fragments stored in zd, or performing simulations based on a fragment and then storing the results of analysis back in the fragment. By convention fragments are usually called Frag and a list of fragments is usually called Frags.

Notes:
  • Values for Frag.Stage:
    • "grow": frag is new or has been grown from a previous frag and is flagged for simulation
    • "assem": frag has been assembled from previous frags and is flagged for simulation
    • "done": frag has finished being simulated; may be used for future growth/assembly
  • Conventions:
    • Note that all FragDataClass member functions return information based on fragment prior restraints only (not global restraints), whereas corresponding functions in zamDataClass return information based on purely global restraints
    • Internally, all atom numbers and residue numbers start at 0, but these are changed for output to base 1

Converts a number to an alpha string.

This function is used to name fragment directories when more than one fragment has the same sequence. The first fragment would be, e.g. r1-8a, the second r1-8b, etc.
