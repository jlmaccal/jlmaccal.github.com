Module zamchoice

source code

Module to automatically decide what to do next during a ZAM run.

zam.py and zammain.py call GetNextActions to prepare a new set of fragments or assemblies to run.

Assemble(MaxChoices, MaxFrags, zd, cd, sd, MaxLoop=None, Weighted=True, Slop=8)

source code  

Make new fragments to assemble based on fragment scores.

This function takes all pairwise combinations of fragments and eliminates
the following:
    * Overlapping fragments
    * Fragments with loops longer than MaxLoop
    * Fragments where we already have a fragment with both parents

The results are then sorted and MaxChoices are printed to the file
fragchoices.txt. Next, new fragments are created for the top MaxFrags
assemblies. The ends of the fragments are corrected so that they 
stretch to the end of the sequence if within Slop of it. Otherwise
some fragments would never reach full length.

Input:
    MaxChoices: Maximum number of choices to list in file
    MaxFrags: Maximum number of new fragments to create
    zd: ZAM database
    cd: contact database
    sd: snapshot database
    MaxLoop: Maximum length of loop allowed. Defaults to MaxLoopDflt
    Weighted: If true then weight scores by snapshot weights,
              otherwise choose maximum.
    Slop: Extend fragment to end of sequence when within Slop

Output: A list of new fragments created.

GetNextActions(zd, cd, sd, MaxFrags=None, MaxChoices=None, AssembleByFrag=None)

source code  

Creates new fragments for growth and/or assembly.

Input:
     zd: ZAM database    
     cd: contact database
     sd: snapshot database
     MaxFrags: Maximum number of new fragments to create
     MaxChoices: Maximum number of assembly choices
     AssembleByFrag: Assemble by fragment rather than by snapshot

Output: a status code in string form.
     "stuck" - simulation cannot proceed any further
     "done" - simulation is done
     "run" - run new frags
     "stop" - user has indicated to stop running new fragments
